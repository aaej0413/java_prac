인터페이스의 장점
1. 개발 시간을 단축시킬 수 있다.
2. 표준화가 가능하다.
3. 서로 관계없는 클래스들에게 관계를 맺어줄 수 있다.
4. 독립적인 프로그래밍이 가능하다.

디폴드 메서드와 static 메서드
    - 원래 인터페이스에는 추상 메서드만 선언할 수 있는데, JDK1.8부터는 디폴트 메서드와 static메서드도 추가할 수 있다.
    - 조상 클래스에 새로운 메서드를 추가하는건 별 일 아니지만,
    - 인터페이스의 경우에는 인터페이스를 구현한 기존의 모든 클래스들이 새로 추가된 메서드를 새로 구현해야한다.
    - 그래서 디폴트 메서드라는 것이 나오게 됨!
    - 디폴트 메서드는 추상 메서드가 아니기 때문에 디폴트 메서드가 추가 되어도 기존 클래스에서 새로 구현하지 않아도 된다.
    - 사용방법은 메서드 앞에 default만 붙이면 됨. 단, 일반 메서드처럼 구현부 몸통{}이 있어야 한다.
